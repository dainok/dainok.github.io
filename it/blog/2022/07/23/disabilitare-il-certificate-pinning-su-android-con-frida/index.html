Disabilitare il Certificate Pinning su Android con Frida

Andrea Dainese
23 Luglio 2022
Sicurezza Personale

Durante le attività di valutazione della sicurezza delle applicazioni Smarthome, è spesso necessario analizzare le comunicazioni HTTPS crittografate. Le applicazioni non solo verificano la validità dei certificati, ma controllano anche che questi siano firmati da un’autorità specifica. Questo meccanismo è chiamato “Certificate Pinning”; sebbene non sia raccomandato, è ampiamente utilizzato:

Attenzione: Il Certificate Pinning non è consigliato per le applicazioni Android a causa dell’elevato rischio legato a modifiche future nella configurazione del server, come il cambio dell’Autorità di Certificazione. Questo potrebbe impedire all’applicazione di connettersi al server senza un aggiornamento software lato client.

Per decifrare una connessione HTTPS protetta dal Certificate Pinning, è necessario adottare un approccio alternativo: Frida, un toolkit di strumentazione dinamica per sviluppatori, reverse-engineer e ricercatori di sicurezza.

Installazione di Frida su Kali Linux

Installazione di Frida su Kali Linux

sudo apt install android-tools-adb android-tools-fastboot

Installeremo una versione precedente di Python dai sorgenti:

sudo apt install libssl-dev libsqlite3-dev liblzma-dev libffi-dev zlib1g-dev
wget https://www.python.org/ftp/python/3.7.12/Python-3.7.12.tgz
tar xzvf Python-3.7.12.tgz -o -C /opt
cd /opt/Python-3.7.12
./configure --enable-optimizations
make

Infine, creiamo un ambiente virtuale:

./python -m venv ~/.venv-frida

Utilizzando l’interprete Python e l’ambiente virtuale appena creati, installiamo Frida:

source ~/.venv-frida/bin/activate
pip install frida-tools objection

Installazione del server Frida su un dispositivo Android

Su un dispositivo Android con permessi di root, abilitare la modalità Debug USB. Andare su Impostazioni > Info sul telefono > Informazioni software e toccare 7 volte su Build Number, poi accedere a Impostazioni > Opzioni sviluppatore > Abilita > Debug USB e attivarlo.

Collegare il telefono al computer tramite cavo USB e riavviare ADB:

adb kill-server
adb devices
- daemon not running; starting now at tcp:5037
- daemon started successfully
List of devices attached
520011a1a0123143        device

Scaricare la versione corretta di Frida e frida-android-unpinning uno script aggiuntivo:

wget https://github.com/frida/frida/releases/download/15.2.2/frida-server-15.2.2-android-arm64.xz
wget https://raw.githubusercontent.com/httptoolkit/frida-android-unpinning/main/frida-script.js

Caricare il server Frida sul dispositivo Android:

unxz frida-server-15.1.12-android-arm.xz
mv frida-server-15.1.12-android-arm frida-server
adb push frida-server /data/local/tmp/

Avviare il server Frida sul dispositivo Android:

adb shell
su
/data/local/tmp/frida-server

Aprire un altro terminale su Kali Linux, attivare l’ambiente virtuale Python creato per Frida e verificare la connessione tra il client (Kali Linux) e il server (Android):

frida-ps -U

Il comando restituirà l’elenco dei processi in esecuzione sul dispositivo Android.

Eseguire un’applicazione senza Certificate Pinning

Ora possiamo eseguire un’applicazione disabilitando il Certificate Pinning. Tutti i comandi seguenti devono essere eseguiti su Kali Linux.

Identificare il nome del pacchetto dell’app da analizzare:

adb shell pm list packages --user 0 -u | grep -i appname

Assicurarsi che l’app non sia già in esecuzione sul dispositivo Android:

frida-ps -U | grep -i AppName

Avviare l’applicazione:

frida --no-pause -U -l ./frida-script.js -f com.appname.start

L’app si avvierà sullo smartphone e apparirà sullo schermo. Tutte le connessioni HTTPS avranno il Certificate Pinning disabilitato e potranno essere intercettate da un MITM Proxy.
