Ansible-Inventar

Andrea Dainese
30 September 2023
Automatisierung

In diesem Artikel werden die beiden Methoden zur Definition eines von Ansible verwendbaren Inventars erkundet. Das Inventar dient als die Quelle der Wahrheit, die die Informationen enthält, die Ansible verwenden wird, um sich mit Geräten zu verbinden. Das Inventar ermöglicht auch die Definition von Variablen auf Host- oder Gruppenebene. Insbesondere können Variablen in fortgeschrittenen Szenarien verwendet werden, um alle Infrastrukturparameter zu definieren, wie beispielsweise Schnittstellen, IP-Adressen, Router-IDs…

Ansible unterstützt sowohl statische als auch dynamische Inventare. Das statische Inventar wird durch eine Datei im INI- oder YAML-Format definiert, während das dynamische Inventar ein ausführbares Programm ist, das JSON auf die Standardausgabe druckt.

Statisches Inventar im INI-Format

Das statische Inventar im INI-Format folgt der folgenden Struktur:

[all]
r1.example.com ansible_host=169.254.1.21

[routers]
r[1:9].example.com

[routers:vars]
ansible_user=admin
ansible_ssh_pass=cisco
ansible_connection=ansible.netcommon.network_cli
ansible_network_os=cisco.ios.ios

Statisches Inventar im YAML-Format

Das statische Inventar im YAML-Format folgt der folgenden Struktur:

all:
  hosts:
    r1.example.com:
      ansible_host: 169.254.1.21
routers:
  hosts:
    r[1:9].example.com:
  vars:
    ansible_user: admin
    ansible_ssh_pass: cisco
    ansible_connection: ansible.netcommon.network_cli
    ansible_network_os: cisco.ios.ios

Dynamisches Inventar im JSON-Format

Wie bereits beschrieben, muss das dynamische Inventar das JSON-Format gemäß der folgenden Struktur verwenden:

{
  "_meta": {
    "hostvars": {
     "r1.example.com": {
        "ansible_host": "169.254.1.21"
      }
    }
  },
  "routers": {
    "hosts": [
      "r1.example.com"
    ],
    "vars": {
      "ansible_connection": "ansible.netcommon.network_cli",
      "ansible_network_os": "cisco.ios.ios",
      "ansible_ssh_pass": "cisco",
      "ansible_user": "admin"
    }
  }
}
