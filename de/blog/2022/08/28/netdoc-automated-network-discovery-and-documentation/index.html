NetDoc: automated network discovery and documentation

Andrea Dainese
28 August 2022
Automatisierung

Vor Jahren arbeitete ich an einem Projekt zur Migration eines Rechenzentrums. Ich war verantwortlich für die Überprüfung der Netzwerkkonfiguration und die Automatisierung der Schritte für Konfiguration, Tests und Migration. Es war nicht möglich, Dutzende von Switches manuell zu überprüfen, also schrieb ich einige Python-Skripte , um Nachbarschafts- und Konfigurationsinformationen von den Switches abzurufen und Visio-Diagramme zu generieren. Am Ende konnte ich eine gesamte Netzwerktopologie in wenigen Minuten entdecken.

NetDoc wäre das industrialisierte Open-Source-Tool für die Multi-Vendor-Netzwerkerkennung. Es basiert auf Netbox, Netmiko, nornir, ntc-templates und netbox-topologyi-views.

Ein nächster Schritt wäre die Integration von draw.io in netbox-topology-views, um Diagramme in einem wiederverwendbaren Format zu exportieren.

Installation

NetDoc erfordert eine funktionierende Instanz von NetBox. Siehe wie man NetBox installiert, dann siehe wie man NetDoc installiert. In diesem Beitrag verwende ich NetBox 3.3.2.

Falls du es nützlich findest, vergiss nicht, es zu unterstützen.

Verwendung

Nach der Anmeldung müssen wir eine Site erstellen. Dann gehen wir zu Plugins -> Netdoc -> Credentials und fügen alle Anmeldeinformationen hinzu oder importieren sie, die für den Zugriff auf die Netzwerkgeräte verwendet werden. Wir können auch per CSV-Import mit folgendem Format importieren:

name,username,password,enable_password
ssh-admin-w-enable,admin,C1sco123,C1sco123
ssh-admin-wo-enable,admin,C1sco123,

Dann müssen wir mindestens ein discoverable hinzufügen/importieren. Ein discoverable ist ein Netzwerkknoten, von dem NetDoc Informationen abrufen kann. Auch hier können wir per CSV mit folgendem Format importieren:

address,credential,mode,site
172.25.82.38,ssh-admin-w-enable,netmiko_cisco_nxos,Test Site
172.25.82.37,ssh-admin-w-enable,netmiko_cisco_nxos,Test Site
172.25.82.36,ssh-admin-w-enable,netmiko_cisco_nxos,Test Site
172.25.82.35,ssh-admin-w-enable,netmiko_cisco_nxos,Test Site

Wir müssen diese Geräte bearbeiten und sie discoverable machen. Jetzt können wir die Erkennung starten, indem wir ein oder mehrere discoverables auswählen und auf den Button klicken.

Die Erkennungsskripte sollten die folgenden Elemente in NetBox auffüllen:

  • Devices
  • Manufacturers
  • Interfaces
  • Cables
  • IP Addresses
  • Prefixes
  • VRFs
  • VLANs

Zusätzlich werden neu erkannte discoverables über CDP/LLDP-Protokolle automatisch hinzugefügt: Sie können nach Überprüfung der IP-Adresse und der Erkennungsmethode entdeckt werden.

Falls ein Gerät nicht automatisch einem discoverable zugeordnet wird (z. B. Cisco XR), kann dies manuell erfolgen.

Zusätzliche Informationen

NetDoc erkennt auch die ARP-Tabelle, die MAC-Adresstabelle und die Routing-Tabelle jedes discoverables.

Schließlich werden alle von den discoverables abgerufenen Daten (Logs) gespeichert und können eingesehen werden:

  • Das Configuration-Flag wird gesetzt, wenn das Log die Konfiguration des discoverables enthält.
  • Das Success-Flag wird gesetzt, wenn der Befehl eine gültige Ausgabe geliefert hat.
  • Das Parsed-Flag wird gesetzt, wenn die Ausgabe erfolgreich analysiert wurde.
  • Das Ingested-Flag wird gesetzt, wenn die analysierte Ausgabe in NetBox eingefügt wurde.

L2-Netztopologie

Mit CDP und LLDP kann NetDoc L2-Adjazenzen entdecken. Für jede Adjazenz wird ein Kabel erstellt:

Mit dem Plugin netbox-topologyi-views können wir L2-Topologien automatisch zeichnen. Jedes Gerät kann mit einem spezifischen Bild verknüpft werden, indem das Attribut Device Role verwendet wird. Gerätegruppen müssen mit einem der folgenden Slugs erstellt werden:

  • access-switch
  • backup
  • core-switch
  • distribution-switch
  • firewall
  • internal-switch
  • isp-cpe-material
  • non-racked-devices
  • power-units
  • role-unknown
  • router
  • server
  • storage
  • wan-network
  • wireless-ap

Durch Öffnen von Plugins -> Topology Views können wir ein L2-Topologiediagramm anzeigen:

Conclusions

NetDoc befindet sich in der Alpha-Version. Derzeit werden nur Cisco-Geräte unterstützt, aber es kann erweitert werden. Auch wenn es sich um eine Alpha-Version handelt, spart es mir bereits viel Zeit.

Wenn du beitragen möchtest, schick mir eine Nachricht. Ich betrachte mich nicht als Softwareentwickler, und es gibt (mit Sicherheit) viele Bugs.
