Frida-Server auf einem Android-Gerät installieren

Andrea Dainese
23 July 2022
Persönliche Sicherheit

Während der Sicherheitsbewertung von Smarthome-Anwendungen muss ich häufig verschlüsselte HTTPS-Kommunikation analysieren. Anwendungen überprüfen in der Regel nicht nur die Gültigkeit von Zertifikaten, sondern auch, ob diese von einer bestimmten Zertifizierungsstelle signiert wurden. Dies wird als „Certificate Pinning" bezeichnet. Obwohl es nicht empfohlen wird, ist es weit verbreitet:

Achtung: Certificate Pinning wird für Android-Anwendungen nicht empfohlen, da zukünftige Änderungen an der Serverkonfiguration, wie der Wechsel zu einer anderen Zertifizierungsstelle, dazu führen können, dass die Anwendung ohne ein Client-Software-Update keine Verbindung mehr zum Server herstellen kann.

Um eine durch Certificate Pinning geschützte HTTPS-Verbindung zu entschlüsseln, benötigen wir einen anderen Ansatz: Frida, ein dynamisches Instrumentierungs-Toolkit für Entwickler, Reverse-Engineers und Sicherheitsexperten.

Frida auf Kali Linux installieren

Zunächst bereiten wir Kali Linux vor:

sudo apt install android-tools-adb android-tools-fastboot

Wir installieren eine ältere Python-Version aus den Quellen:

sudo apt install libssl-dev libsqlite3-dev liblzma-dev libffi-dev zlib1g-dev
wget https://www.python.org/ftp/python/3.7.12/Python-3.7.12.tgz
tar xzvf Python-3.7.12.tgz -o -C /opt
cd /opt/Python-3.7.12
./configure --enable-optimizations
make

Schließlich erstellen wir eine virtuelle Umgebung:

./python -m venv ~/.venv-frida

Mit dem oben erstellten Python-Interpreter und der virtuellen Umgebung können wir Frida installieren:

source ~/.venv-frida/bin/activate
pip install frida-tools objection

Frida-Server auf einem Android-Gerät installieren

Auf einem gerooteten Android-Gerät den USB-Debugging-Modus aktivieren. Gehe zu Einstellungen > Über das Telefon > Softwareinformationen und tippe siebenmal auf Build-Nummer, dann navigiere zu Einstellungen > Entwickleroptionen > Aktivieren > USB-Debugging und schalte es ein.

Verbinde das Telefon über ein USB-Kabel mit dem Computer und starte ADB neu:

adb kill-server
adb devices
- daemon not running; starting now at tcp:5037
- daemon started successfully
List of devices attached
520011a1a0123143        device

Lade die passende Frida-Version und ein zusätzliches Script herunter:

wget https://github.com/frida/frida/releases/download/15.2.2/frida-server-15.2.2-android-arm64.xz
wget https://raw.githubusercontent.com/httptoolkit/frida-android-unpinning/main/frida-script.js

Übertrage den Frida-Server auf das Android-Gerät:

unxz frida-server-15.1.12-android-arm.xz
mv frida-server-15.1.12-android-arm frida-server
adb push frida-server /data/local/tmp/

Starte abschließend den Frida-Server auf dem Android-Gerät:

adb shell
su
/data/local/tmp/frida-server

Öffne ein weiteres Terminal auf Kali Linux, aktiviere die für Frida erstellte Python-Umgebung und überprüfe die Netzwerkverbindung zwischen dem Client (Kali Linux) und dem Server (Android):

frida-ps -U

Der letzte Befehl sollte die Liste der laufenden Prozesse auf dem Android-Gerät zurückgeben.

Eine Anwendung ohne Certificate Pinning ausführen

Jetzt können wir eine Anwendung ausführen und das Certificate Pinning deaktivieren. Alle folgenden Befehle werden auf Kali Linux ausgeführt.

Den Paketnamen der zu analysierenden Anwendung ermitteln:

adb shell pm list packages --user 0 -u | grep -i appname

Sicherstellen, dass die Anwendung nicht bereits auf dem Android-Gerät läuft:

frida-ps -U | grep -i AppName

Die Anwendung starten:

frida --no-pause -U -l ./frida-script.js -f com.appname.start

Die App wird auf dem Smartphone gestartet und auf dem Bildschirm angezeigt. Alle HTTPS-Verbindungen haben nun das Certificate Pinning deaktiviert und können von einem MITM-Proxy abgefangen werden.
